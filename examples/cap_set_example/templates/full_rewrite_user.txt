You are allowed and encouraged to produce a complete, runnable Python module
that constructs a large cap set in F_3^n.

IMPORTANT: For this task, fix the dimension to n = 6.

A cap set is a subset A of F_3^n such that there are no three *distinct*
elements x, y, z in A with x + y + z â‰¡ 0 (mod 3) coordinate-wise.

Your goal is to maximize |A|. Correctness is mandatory; larger sets score higher.

## REQUIREMENTS

- Output at least one COMPLETE, runnable Python file.
- The file must define:

    def generate_set(n) -> List[Tuple[int, ...]]

  which returns a cap set with entries in 0/1/2.

- Only Python standard library imports are allowed.

- If randomness is used:
  - Define a module-level constant `_RANDOM_SEED`
  - Seed all RNGs deterministically

## ALLOWED STRATEGIES (NOT EXHAUSTIVE)

You are NOT restricted to greedy algorithms.

You are encouraged to explore:
- Local search / hill climbing
- Randomized multi-start construction
- Simulated annealing
- Genetic or evolutionary recombination
- Constraint-based incremental construction
- Compact rule-based or priority-based generators
- Hybrid methods combining greedy + random perturbation

Programs that *encode structure concisely* (rather than explicitly listing
all vectors) are especially encouraged.

## IMPLEMENTATION CONSTRAINTS (IMPORTANT)

- Keep runtime bounded and predictable.
- Avoid super-exponential or unbounded loops.
- If your approach is potentially expensive, introduce explicit limits such as:

    MAX_ITERATIONS
    MAX_RESTARTS
    TIME_LIMIT_SECONDS
    MAX_CANDIDATES

- Prefer incremental or local checks over full O(|A|^3) verification.
  Useful ideas include:
  - Precomputing forbidden third points
  - Encoding vectors as base-3 integers
  - Maintaining conflict counters
  - Sampling candidate moves

- When unsure, prefer a SMALL, SAFE modification over a full rewrite.
